## 一、项目概述
基于上市公司十大流通股东数据与股票历史行情数据，构建针对**自然人股东（牛散）**的完整持仓历史、成本收益及分红记录分析功能。系统需支持：
1. 生成每个自然人股东**所有涉及股票**的**连续报告期持仓记录**（含零持仓补全），并计算**累计成本、平均成本、浮动盈亏、至今涨幅**等指标。
2. 结合**分红事件表**，计算每个自然人股东在历次分红中获得的**分红金额及股息率**。

## 三、模块一：自然人股东持仓历史与成本收益计算

### 3.1 业务规则

#### 3.1.1 自然人筛选
- 直接使用 `gdlx = '自然人'` 过滤（若表中实际存储为代码，请相应调整）。

#### 3.1.2 零持仓补全（完整时间序列网格）
- 对于每个 `(dm, gdmc)` 组合，找到其**首次出现在十大股东表**的报告期 `first_date`。
- 生成从 `first_date` 到表中**最大报告期**之间的**所有报告期**记录。
- 未进入前十大的报告期，`cgsl` 和 `cgbl` 填充为 `0`。

#### 3.1.3 价格匹配（处理停牌与节假日）
- 每个报告期 `jzrq` 匹配**最近一个交易日**的有效价格。
- 匹配条件：`hs_stock_history_trading.t <= jzrq` 且 `sf = 0`（非停牌）。
- 价格取值优先级：`c`（收盘价）→ `o`（开盘价）。若均为 `NULL`，则向前沿用该股票上一报告期的有效价格。

#### 3.1.4 成本计算逻辑（加权平均成本 + 清仓重置）

**状态变量（按股东+股票分组，报告期升序迭代）**
- `total_cost`：当前持仓周期的累计投入成本。
- `prev_hold`：上期持股数量。
- `is_holding`：是否处于持仓状态（`hold > 0`）。

**增持处理（`hold > prev_hold`）**
- 买入量 = `hold - prev_hold`
- 买入成本 = 买入量 × 本期匹配到的价格
- `total_cost += 买入成本`

**减持处理（`hold < prev_hold`）**
- 卖出量 = `prev_hold - hold`
- 减持前平均成本 = `total_cost / prev_hold`
- 成本扣减 = 卖出量 × 减持前平均成本
- `total_cost -= 成本扣减`

**清仓与再建仓重置规则**
- 当 `hold = 0` 时：
  - 每股平均成本 `avg_cost` 设为 `NULL`。
  - `total_cost` **保留当前值**（可记录上一周期结束时的累计投入），但 `is_holding` 标记为 `False`。
- 当由 `hold = 0` 变为 `hold > 0` 时：
  - 强制将 `total_cost` 重置为 `0`，视为全新持仓周期。
  - `is_holding` 标记为 `True`。

**每股平均成本**
- 当 `hold > 0` 时：`avg_cost = total_cost / hold`
- 当 `hold[M`F1[M`F1[M`F1 = 0` 时：`avg_cost = NULL`

四、模块二：自然人股东近年分红收益明细
4.1 业务规则
4.1.1 分红权益归属判断
判断依据：自然人股东在分红事件的股权登记日（edate）是否持有该股票。

由于十大股东数据为季度末快照，采用最近一次报告期持仓作为近似：

对于每次分红，取 jzrq <= edate 的最近一条持仓记录，若该记录 cgsl > 0，则认为享有本次分红。

4.1.2 分红金额计算
每股分红 = Send / 10（Send 为每10股派息税前金额）

分红总额 = 持股数 × 每股分红

4.1.3 除权日股价计算
取 cdate（除权除息日）当天的行情数据。

股价 = (h + l) / 2（不复权均价）

若当天停牌（sf=1）或无数据，则向前取最近一个交易日的数据。

4.1.4 股息率计算
股息率 = 每股分红 / 除权日股价 × 100%

4.1.5 输出指标
字段名称	计算逻辑 / 来源
股票代码	dm
股票名称	关联 stock_info.name
牛散名称	gdmc
分红日期	cdate
每股分红(元)	Send / 10
持股数量	最近一期 cgsl
分红总额(元)	持股数 × 每股分红
除权日股价	(h + l) / 2
股息率(%)	每股分红 / 除权日股价 × 100%

五、全局注意事项
数据去重：若同一股东在同一报告期对同一股票有多条记录，需先按 dm, gdmc, jzrq 汇总 cgsl 和 cgbl（通常取最大值或求和，根据数据质量决定）。

前复权问题：当前方案使用不复权价格计算成本与股息率。若数据源支持前复权价格，请在 SQL 层替换 c、o、h、l 为前复权字段；否则需在前端标注“基于不复权价格估算”。

股权登记日与季报时间差：模块二采用“最近季报持仓>0”作为享有分红权益的判断，属于合理估算，需在页面注明“分红数据基于最近季报持仓估算”。

极端价格处理：若价格 ≤ 0 或 > 10000，视为无效，沿用前序有效价格。

性能考虑：数据量较大时，建议对 jzrq、t、edate 等日期字段建立索引，SQL 中使用分区或物化视图优化。
