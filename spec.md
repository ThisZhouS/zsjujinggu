## 1. 项目概述

### 1.1 项目简介

King 财经数据平台是一个面向中国股票市场的专业数据服务平台，专注于跟踪知名投资者（牛散）的持仓变化、分析涨幅榜数据、提供财经资讯。平台采用前后端分离架构，为用户提供实时、准确、可视化的投资数据分析。

### 1.2 核心价值主张

- **牛散跟踪**: 实时跟踪知名投资者的持仓变动，包括增持、减持、新进股票
- **涨幅分析**: 提供多周期涨幅榜、涨停板数据，帮助用户发现市场热点
- **数据可视化**: 通过图表直观展示持仓分布、收益走势等关键数据
- **专业资讯**: 提供巴菲特、木头姐等知名投资者的持仓分析和财经文章

### 1.3 目标用户

| 用户类型 | 核心需求 | 付费意愿 |
|---------|---------|---------|
| 散户投资者 | 跟踪牛散持仓、发现投资机会 | 中等 |
| 量化开发者 | 获取结构化数据、API 接口 | 高 |
| 财经媒体 | 获取行业数据、写作素材 | 中等 |
| 机构投资者 | 市场情绪分析、热点追踪 | 高 |

#### 2.1.2 持仓分析

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|-------|
| F-004 | 增持榜单 | 展示牛散增持股票列表，按增持市值排序 | P0 |
| F-005 | 减持榜单 | 展示牛散减持股票列表，按减持市值排序，标记清仓股票 | P0 |
| F-006 | 新进榜单 | 展示牛散新进股票列表，按新进市值排序 | P0 |
| F-007 | 共同持仓 | 支持选择多个牛散，计算其共同持仓股票 | P0 |
| F-008 | 十大增持 | 展示全市场增持市值 Top 100 记录 | P1 |

**计算规则**:
- 增持数量 = 本期持股数 - 上期持股数（本期 > 上期）
- 减持数量 = 本期持股数 - 上期持股数（本期 < 上期）
- 新进判断: 上期不存在或上期持股数为 0
- 清仓标记: 本期持股数为 0

## 3. 技术需求

### 3.1 架构要求

| 需求编号 | 需求描述 | 约束条件 |
|---------|---------|---------|
| T-001 | 前后端分离架构 | 前端使用 Next.js 14，后端使用 NestJS 10 |
| T-002 | 严格 3 层架构 | Controller → Service → Repository → Database，禁止跨层调用 |
| T-003 | Monorepo 结构 | 使用 pnpm workspace 管理多包项目 |
| T-004 | 共享类型定义 | 前后端共用 Prisma Schema 和 TypeScript 类型 |

### 3.2 技术栈要求

| 层级 | 技术 | 版本约束 |
|-----|------|---------|
| 前端框架 | Next.js (App Router) | 14.x |
| UI 框架 | Ant Design | 5.x |
| 图表库 | ECharts | 5.x |
| 数据获取 | SWR | 2.x |
| 后端框架 | NestJS | 10.x |
| ORM | Prisma | 5.x |
| 数据库 | PostgreSQL | 15+ |
| 缓存 | Redis | 7.x |
| 包管理器 | pnpm | 8.x |

### 3.3 编码规范要求

| 规则编号 | 规则描述 | 严重程度 |
|---------|---------|---------|
| R-001 | Decimal/BigInt 必须转换为 number 后运算 | 严重 |
| R-002 | 可空字段必须守卫（?? 或 if (value != null)） | 严重 |
| R-003 | redis.mget() 返回数组，用索引访问 | 严重 |
| R-004 | 统一响应格式 { code, message, data } | 严重 |
| R-005 | VIP 拦截返回 HTTP 200 + code:403 | 严重 |
| R-006 | 外部 API 失败返回缓存或空数组 | 严重 |
| R-007 | 所有 /admin/* 路由需 JwtAuthGuard + AdminGuard | 严重 |
| R-008 | IDOR 防护：Service 层验证资源所有权 | 严重 |
| R-009 | bcrypt.hash(password, 10) 必须传 saltRounds | 严重 |
| R-010 | API Key 只存 hash，验证时 hash 后比对 | 严重 |

### 3.4 数据模型要求

## 4. 非功能性需求

### 4.1 性能要求

| 指标 | 目标值 | 测量方式 |
|-----|-------|---------|
| API 响应时间 P95 | < 500ms | 不含外部 API 调用的接口 |
| API 响应时间 P99 | < 2000ms | 包含外部 API 调用的接口 |
| 首页加载时间 | < 3s | 3G 网络环境 |
| 列表页渲染时间 | < 1s | 100 条数据 |
| 数据同步全量 | < 30 分钟 | 股票列表+行情+股东+分红 |

### 4.2 可用性要求

| 指标 | 目标值 | 实现方式 |
|-----|-------|---------|
| 系统可用性 | ≥ 99.5% | Docker Compose 部署 |
| 缓存可用性 | 降级为数据库查询 | Redis 不可用时自动降级 |
| 外部 API 降级 | < 3s | 超时后返回缓存或空数组 |
| 并发支持 | ≥ 100 同时在线用户 | 单实例部署 |

### 4.3 安全要求

| 要求 | 实现方式 |
|-----|---------|
| 认证 | JWT Token + Passport Strategy |
| 授权 | Guard 层（JwtAuthGuard, AdminGuard, VipGuard） |
| 密码安全 | bcrypt hash，saltRounds=10 |
| API Key 安全 | 存储 bcrypt hash，仅前缀可见 |
| IDOR 防护 | Service 层验证 resource.userId === currentUser.id |
| 敏感数据脱敏 | 手机号 138****1234，API Key 前 8 位 |
| SQL 注入防护 | Prisma ORM 参数化查询 |
| XSS 防护 | Next.js 自动转义，富文本 DOMPurify |

### 4.4 数据质量要求

| 要求 | 实现方式 |
|-----|---------|
| 数据一致性 | Prisma 事务、唯一约束 |
| 数据精度 | Decimal(18,2) 金融精度，BigInt 持股数 |
| 数据完整性 | 外键约束、级联删除 |
| 数据清洗 | sanitizeCount/sanitizePrice/sanitizeRatio |
| ST 股过滤 | 涨幅榜排除名称含 ST 的股票 |

## 8. 验收标准摘要

核心验收标准（详见 checklist.md）：

1. **数据准确性**: 牛散持仓数据与数据源一致，计算公式正确
2. **功能完整性**: 所有 P0 功能可用，交互流程顺畅
3. **性能达标**: API 响应时间符合 4.1 节要求
4. **安全合规**: 通过安全扫描，无高危漏洞
5. **兼容性**: 主流浏览器测试通过
6. **文档齐全**: 部署文档、API 文档完整
